게임 이름: 야생 대결 (Wild Duel)

[게임 개요]
- 모든 플레이어는 게임 시작 시 지정된 월드 스폰에서 시작
- 시작 아이템: 돌 도끼, 돌 곡괭이
- 게임은 총 2단계로 구성됨
  1. 파밍(준비) 시간: 15분, PvP 비활성화, 죽어도 부활 가능, keepInventory = true
  2. 전투 시간: PvP 가능, 죽으면 관전 모드, keepInventory = false, 월드보더 축소 시작

[명령어 설명]

1. /wd set
- 게임 시작 전 대기 상태 진입
- 준비장소 생성 (현재 월드에 임시 구조물 등)
- 현재 월드의 스폰 위치를 준비장소로 설정
- 모든 플레이어에게 '포화' 효과 부여 (움직임 제한 목적)

2. /wd sp
- 명령어를 입력한 플레이어의 현재 위치를 "게임 시작 시 스폰 위치"로 등록
- 게임 시작 시 이 위치가 월드 스폰으로 설정됨

3. /wd start
- 게임 시작
- 모든 플레이어를 등록된 시작 위치(월드 스폰)로 텔레포트
- 기본 아이템(돌 도끼, 돌 곡괭이) 지급
- keepInventory true 설정
- 팀 설정 및 PvP 비활성화
- 월드보더 반지름 1000으로 설정
- 파밍(준비) 시간 15분 시작
- 타이머는 보스바로 표시
- 타이머 종료 시:
  → 팀 해체, PvP 활성화  
  → keepInventory false  
  → 사망 시 관전 모드
  → 월드보더 축소 시작

4. /wd ts <초>
- 파밍(준비) 시간이 진행 중일 때 사용 가능
- 남은 준비 시간(초 단위)을 임의로 설정

[게임 진행 요약]

1. /wd set → 플레이어 대기 + 준비장소 생성 + 포화 효과  
2. /wd sp → 게임 시작 위치 등록  
3. /wd start → 게임 시작 (파밍 15분)  
4. /wd ts <초> → 준비 시간 강제 조정  
5. 파밍 종료 후 전투 시작

[기타 시스템]
- 준비 시간 중: 사망 시 자동 부활, keepInventory = true
- 전투 시간 중: 사망 시 관전 모드, keepInventory = false
- 전투 시작 시 월드보더가 반지름 50까지 서서히 축소됨
- 관전자 채팅 또는 자동 재시작 시스템 등 추가 확장 가능

[예상 에러/예외 상황]
- `/wd ts`에 음수나 숫자 외 문자가 입력될 때  
  → NumberFormatException 발생 → 입력값 파싱 전 `try-catch` 및 유효 범위(>=0) 검증 후 안내  
- `/wd set` 또는 `/wd start` 전에 lobbySpawn/worldSpawn이 등록되지 않을 때  
  → NullPointerException 발생 → 명령 실행 전 등록 상태 확인, 미등록 시 오류 메시지 출력  
- 게임 진행 중(`/wd start` 이후) 다시 `/wd start` 실행 시  
  → IllegalStateException 발생 → `gameState` 검사로 중복 시작 방지  
- 월드보더 축소 작업 중 잘못된 월드 참조 시  
  → IllegalArgumentException 발생 → 월드 일치 여부 확인 후 처리  
- Scoreboard 팀 생성 또는 `friendlyFire` 설정 실패 시  
  → NullPointerException 발생 → Scoreboard 초기화 시점 검증  
- 스케줄러(BukkitTask) 취소 또는 미등록 시  
  → Scheduler 동작 불가 → 반환된 Task 객체 저장·검증 로직 추가  
- `prepTimeSeconds`를 음수로 변경할 때  
  → 논리 오류 발생 → 파라미터 변경 전 최소값(0) 검증
